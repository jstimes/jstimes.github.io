PriceFinder

8 Aug, 2016 · Read in about 2 min · (332 Words)

PriceFinder is a project I started working on in Spring 2016, and stopped working on in August 2016. The main idea of the app is to let users search for products they are interested in. Results are pulled in from a variety of sites using their API (Amazon, Best Buy, Walmart, etc.). With these results, users select the ones they would consider purchasing (e.g. maybe the user doesn’t want to buy from eBay), and give a target prices they would like to buy at. Those items are saved, and re-searched for on a daily basis to see if any of their prices have fallen below the target price. This allows them to only use the app when they have a new product they desire, and be notified when it is affordable for them.

Users can browse through their saved searches, and update the target price, or delete specific items. Additionally, each individual item has a link to its site so the user can buy it directly without having to back out of the app, open a browser, and search for it again. There was a recommended tab where I hoped to provide suggestions based on similar users, but have not implemented it as I ended up not using a server to store data and instead use the device’s local storage.

The Android version is functional, but has a lot of UI work still to be done. As I was wrapping up the iOS component, one of the last things I was doing was incorporating the Amazon API (one of the best online vendors and has support for retrieving recommended items). I discovered that their terms of service agreement does not allow any mobile app to display their Product Advertising content without written consent. Upon discovering that, I looked into the other API terms’ of service I was using and they have some disappointing requirements as well. Until I find a workaround/get written consent from Amazon, this project is being put on hold.
